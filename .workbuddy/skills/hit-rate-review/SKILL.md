# 命中率复盘（Hit-Rate Review）

选题罗盘（TopicCompass）系统的**数据回流复盘环节**。把创作者发布后回流的互动数据，转化成一份**可行动的复盘周报**——让系统"越用越准"，也让创作者清楚"下周该往哪使劲"。

## 什么时候用

- 用户要做内容复盘 / 周报 / 效果总结。
- 想知道最近哪些内容表现好、规律是什么。
- 作为**每周 Automation 定时任务**自动生成周报。

## 数据来源（选题罗盘系统）

选题罗盘部署在 EdgeOne Makers（域名 `topic.yeranyang.com`），提供：

- `GET /api/stats` —— 命中率总览 + 各维度规律（关键词/时段/标题特征命中率排名、按选题排名）。**复盘主数据源。**
- `GET /api/posts` —— 历史发布明细（需要举具体例子时取样）。

调用后拿到 JSON，你负责把它翻译成人话周报。若拿不到线上数据，请用户提供近期发布数据（选题/标题/时段/互动指标），你本地按同样口径分析。

## 命中口径（与系统一致）

- **互动热度分** = `阅读×0.01 + 点赞×1 + 在看×2 + 评论×2 + 分享×3 + 涨粉×4`。
- 进入历史 **70 分位** 记为「命中/爆款」。
- **命中率** = 命中篇数 / 总篇数。

## 工作流程

1. **取数**：调 `GET /api/stats`（默认全量；如需近一周可结合 `/api/posts` 按时间过滤）。
2. **算趋势**：对比本期 vs 上期命中率、平均热度分的变化。
3. **提炼模式**：找出本期"高命中"的关键词、时段、标题特征；也点出"踩雷"的模式。
4. **给建议**：基于规律，给下周 3 条可执行建议（写什么方向、什么时段发、标题怎么起）。

## 输出规范（周报模板）

```
# 内容命中率复盘周报（截至 {日期}）

## 一、核心指标
- 命中率：{本期}%（环比 {↑/↓}{差值}）
- 总产出：{N} 篇 · 命中 {H} 篇
- 平均热度分：{avg}

## 二、有效模式（继续做）
- 选题：{高命中关键词，附命中率}
- 时段：{高命中时段，附命中率}
- 标题：{有效标题特征}

## 三、待改进（少踩雷）
- {低命中的选题/时段/标题特征}

## 四、下周建议（可执行）
1. ...
2. ...
3. ...
```

## 注意事项

- 数字必须来自真实回流数据，不虚构；样本少时标注"置信度有限"。
- 复盘要落到"下一步动作"，不能只罗列数据。
- 作为 Automation 运行时，输出即为周报正文，可直接推送/存档。
- 语言与用户一致，默认中文。
