# 分支模型与安全钩子（贡献指南）

本项目采用 **develop 为集成主干** 的分支流程，并通过 Git pre-push 钩子拦截敏感信息泄露。**克隆仓库后请先执行「一次性启用」。**

---

## 一、一次性启用（clone 后必做）

Git 默认不会自动应用仓库内的钩子，需手动把 hooks 路径指向 `.githooks/`：

```bash
git config core.hooksPath .githooks
chmod +x .githooks/*      # macOS / Linux
```

执行后，本仓库的所有钩子即对你本地生效。

---

## 二、分支模型

| 分支 | 角色 | 部署环境 | 谁能直接改 |
| --- | --- | --- | --- |
| `main` | 生产主干 | **生产**（`topic.yeranyang.com`） | 仅通过 develop 合入 |
| `develop` | **集成/预览主干** | **预览**（EdgeOne 自动分配预览 URL） | 仅通过 feature/bugfix 合入 |
| `feature/*` | 新功能 | 预览（推送即建预览） | 开发者 |
| `bugfix/*` | 缺陷修复 | 预览 | 开发者 |

### 流程

```
feature/xxx ──┐
              ├──(合并)──▶ develop ──(验收后合并)──▶ main
bugfix/xxx  ──┘             预览环境                  生产环境
```

1. 从 `develop` 切出工作分支：`git switch -c feature/xxx develop`
2. 开发、提交，推送（触发预览构建自检钩子）
3. 合并回 `develop`：验证预览环境
4. 预览验收通过后，由 `develop` 合并进 `main` 发布生产

> **EdgeOne 多环境**：`main` 独占生产，其余任何分支 push 后自动进入预览环境。因此 `develop` 及各 feature 分支推送后都会得到独立预览 URL。KV 为项目级共享，预览验证请**只读不写**。

## 三、pre-push 敏感信息扫描

`.githooks/pre-push` 会在每次 `git push` 前，扫描**本次待推送提交新增的行**，命中以下任一规则即**拦截推送**：

- GitHub PAT（`github_pat_` / `ghp_` / `ghu_` 等）
- OpenAI 密钥（`sk-...`）
- 腾讯云 SecretId（`AKID...`）、AWS Access Key（`AKIA...`）
- Google API Key（`AIza...`）、Slack Token（`xox...`）
- 私钥文件头（`-----BEGIN ... PRIVATE KEY-----`）
- 通用密钥赋值（`secret/password/api_key/access_token = "..."`）

含 `YOUR_`、`example`、`placeholder`、`<...>` 等占位内容的命中视为安全，不拦截。

### 误报时如何绕过

确认确实是误报（例如示例文档里的假密钥），可临时跳过：

```bash
git push --no-verify
```

> ⚠️ 仅在**确认无真实密钥**时使用。真泄露时正确做法是：移除密钥 → 改用环境变量/密钥管理 → 清除历史（`git filter-repo`）→ 到平台**吊销并轮换**该凭证。

### 新增扫描规则

编辑 `.githooks/pre-push` 顶部 `patterns` 数组，按 `"规则名|正则"` 格式追加即可。

## 四、发布前合规自查清单（对齐腾讯规范）

> 本项目仓库公开（`github.com/guangyang1206/zhiwei-topic-compass`），每次 **push 前** 与 **合并 develop / main 前**，请逐项确认。
> ⚠️ **pre-push 钩子只扫描「本次待推送提交的新增行」，且仅覆盖密钥类规则**——它**不扫描**内部域名/措辞、个人路径、PII。**钩子通过 ≠ 完全合规**，仍需走此清单做人工核对。

### A. 敏感凭据
- [ ] 代码/配置中**无真实密钥**：GitHub PAT（`github_pat_`/`ghp_`/`ghu_`）、OpenAI（`sk-...`）、腾讯云（`AKID...`）、AWS（`AKIA...`）、Google（`AIza...`）、Slack（`xox...`）、私钥文件头（`-----BEGIN ... PRIVATE KEY-----`）、通用 `secret/password/api_key/token = "..."`
- [ ] 所有密钥走**环境变量 / 密钥管理**，不落库（`.env`、`*token*.env` 已在 `.gitignore`）
- [ ] 若曾误提交真实密钥：已到平台**吊销并轮换**该凭证，并用 `git filter-repo` 清除历史

### B. 公司内部信息（腾讯规范重点）
- [ ] **无内网域名**：`woa.com` / `*.oa.com` / `km.woa` / `iwiki` / `venus` 等
- [ ] **无内部系统**作为真实引用：TAPD / iWiki / KM / venus 等
- [ ] **无 BG/部门缩写**作为内部信息：`csig` / `wxg` / `ieg` / `pcg` / `teg` / `cdg`（注意：这些缩写也可能误中普通英文词，如 `teg` 含于 `integer`——人工核对时认词义，不要机械匹配）
- [ ] **无工号 / 员工号 / 真实姓名 / 内部组织架构**
- [ ] **无影射具体雇主的措辞**：如「在腾讯做…」「大厂」等；统一用中性表述（如「在一家科技公司做…」）

### C. 个人与路径信息
- [ ] **无硬编码本地绝对路径**：`/Users/xxx` / `C:\Users\xxx` / 机器专属路径（会泄露用户名且他人无法复现构建）
- [ ] **无个人邮箱 / 手机号 / 身份证号**

### D. 依赖与可复现
- [ ] 依赖通过 `package.json` / lockfile 声明，**不用硬编码绝对路径 import**
- [ ] `npm install` 后任意机器可构建，无本地 workspace 路径依赖（如本仓库 `build-docs.mjs` 已改用裸包名 `marked`）

### E. 钩子与流程
- [ ] 已执行「一次性启用」（`core.hooksPath`），push 时看到 `[pre-push] ✓ 未发现敏感信息`
- [ ] 自动生成产物（`app/reports/`、`app/.workbuddy/`）已 `gitignore`，**未入库**
- [ ] 合并 `develop` 前预览环境已验证；合并 `main` 前 `develop` 已验收通过

### 手动预检（钩子之外的兜底）

提交前可自己跑一遍与钩子同口径的扫描，覆盖未跟踪文件与全量内容：

```bash
# 凭据类（仅示例规则名/占位符应放行）
git grep -nIE 'github_pat_[A-Za-z0-9]{20,}|ghp_|AKID[A-Za-z0-9]{20,}|sk-[A-Za-z0-9]{20,}|-----BEGIN' -- ':!*.md' ':!*.lock'

# 内部域名 / 系统 / BG 缩写（人工逐条认词义）
git grep -nIiE 'woa\.com|\.oa\.com|tapd|iwiki|km\.woa|venus|csig|wxg|ieg|pcg|teg|cdg|内网|工号|员工号'

# 硬编码本地路径 / 个人身份
git grep -nI '/Users/|C:\\Users|1[3-9][0-9]{9}'
```

> 命中的**示例/规则名/占位符**不算泄露；只有真实凭据或真实内部信息才需要处置。
